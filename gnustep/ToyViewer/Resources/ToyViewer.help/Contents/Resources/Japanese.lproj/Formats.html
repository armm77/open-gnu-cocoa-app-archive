ToyViewer (Ver.4.30)

ToyViewerが表示できる画像形式

以下の形式の画像ファイルは、Finderのインスペクタで設定することで、アイコンをダブルクリックしてオープン可能です。
tiff, pdf, eps, gif, bmp, png, jpg, JPEG2000(jp2, jpc, j2k), bie(jbig), pcx, pcd, pnm, ppm, pbm, pgm, xbm, mag, Sun Raster

JPG形式への保存

JPG形式は高い圧縮率を達成しますが、元の画像の持っていた情報は失われます。JPEGでは一般に、画質を良くすると圧縮は良くなくなり、圧縮率を高めると画質が悪くなります。このパラメータはセーブパネルで設定できます。
カラーあるいはモノクロ 8ビットの連続階調画像への適用を勧めます。それ以外の画像、特に白黒イメージへの適用は画質が劣化するだけでなく、ファイルサイズも大きくなる傾向があります。
保存の際、インタレース(プログレッシヴ)形式が選択できます。 Webページで表示する場合、通信速度が遅くても画像全体の概観が分かるという利点があります。圧縮率は通常と比べてやや悪くなります。

白黒２値画像は bie(jbig)形式として保存すると、サイズを小さくでき、画質の劣化もありません。

TIFFファイルの圧縮形式

tiffファイルへの保存の際、圧縮をしない、またはLZWで圧縮のいずれかを選択できます。LZWはJPEGとは異なり、圧縮によって画質が損なわれることはありません。

OPENSTEPにはJPEG圧縮された TIFFファイル形式が存在しましたが、 Mac OS X ではこの形式はサポートされていません。読み込んで表示することもできません。

JPEG2000形式について

JPEG2000形式はJPEGと同様に画質を良くすると圧縮は良くなくなり、圧縮率を高めると画質が悪くなります。このパラメータはセーブパネルで設定できます。 また、JPEG2000形式は元画像の持つ情報を損なわないで保存することもできます。
カラーあるいはモノクロ8ビットの連続階調画像への適用を勧めます。 損失のない圧縮としては、pngを上回る圧縮率を実現できます。また、損失のある圧縮においても、比較的高い圧縮率でもJPEGに比較して人工的なノイズが現れにくいという特徴があります。

ToyViewerは、QuickTimeやPreviewで読み書き可能なjp2形式に加え、ネット上で利用されているj2k形式(jpc形式)にも対応しています。 なお、ToyViewerは JPEG2000の読み書きに JasPer (http://www.ece.uvic.ca/˜mdadams/jasper/) を使っています。

EPSファイルについて

現在の Mac OS X (10.1.5) はEPSファイルの表示をサポートしていません。Cocoaには関連するAPIが用意されていますが、機能していません。

PDFからビットマップへの変換

「拡大・縮小した画像の作成」を参照して下さい。
メニューの「画像操作 / PDF/EPS→Bitmap」を使うこともできますが、この機能ではEPSのサイズを変更することができません。

BMP形式(DIB形式)ファイルについて

BMP形式のファイルには、標準的なフルカラーの形式、パレットを持つ形式のほかに、15Bitsあるいは16Bitsカラー形式と呼ばれる形式があります。また、さらにこれらを RLEで圧縮した形式もあるようです。Mac OS Xは標準的なBMP形式の表示は可能ですが、その他の形式には対応していません。
ToyViewerは15Bitsカラー形式や、圧縮された形式にも対応しています。

減色処理について

gifファイルへの変換の際、元画像が 256色以上の色を使用していた場合、減色処理を行う必要があります。
減色のアルゴリズムは Paul Heckbertの「メディアン・カット」アルゴリズムを使用しています。これは ppmquant(by Jef Poskanzer)のものと同じですが、ToyViewerは効果的な色の取捨選択を行っているため、大変良好な画像が得られます。
ToyViewerはまた、擬似ハーフトーン法、ディザ法による減色処理も提供しています。「固定パレット」を使うと、Windowsや Macintosh に共通のパレットを使った減色が行えます。

CMYK形式について

tiffファイルの中には、通常のRGBでなく、CMYKによって色を表したものがあります。ToyViewerでは「画像操作/CMYK→RGB」によって RGB表現に変換することができますが、現在のインプリメントでは大変に時間がかかります。

また、CMYK表色系は印刷時のインクの特性に依存するものであるため、RGBへ変換した後の色はソフトウェアによって大きく異なることがあります。

インタレース形式

ToyViewerはインタレース(プログレッシヴ)形式の gif, png, および jpg画像を読み書きできます。インタレース形式は WWW などで画像を転送する際にじわじわと表示させる効果を持ちますが、保存の目的にはあまりメリットはなく、逆にデータサイズが大きくなる傾向があります。
また、古いブラウザなどでインタレースのjpg形式が扱えないことがあります。
